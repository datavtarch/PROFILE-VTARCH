# Frontend UI Design

Create a focused productivity interface, not a marketing page.

## Workflow

1. Read `/docs/ui-guidelines.md`.
2. Design mobile-first, then expand to desktop.
3. Make the first screen useful: show task views and actions after login.
4. Use predictable controls:
   - Icons for add, edit, complete, delete, filter, settings.
   - Tabs or segmented controls for task views.
   - Inputs/selects/date controls for task editing.
5. Keep text compact and scannable. Avoid instructional copy inside the app unless it is needed for an empty state.
6. Verify that long task titles, small screens, and empty states do not break layout.
7. Before finishing UI work, check `/docs/test-checklist.md` for affected flows.
